# Living Hell

**ФИО:** Харитоненко Максим Дмитриевич, Новиков Александр Михайлович  
**SFML:** 3.0.2

## Основная система
**Core Heat** — бездействие, атаки врагов и способности повышают тепло ядра. При достижении максимума — смерть. Основа всего геймплея. (Player.cpp, Abilities.cpp)

## Доп. механика 1
**Система способностей** — игрок получает способности при повышении уровня, каждая взаимодействует с Core Heat по-своему. (Abilities.cpp)

## Доп. механика 2
**Разное поведение врагов** — Чертила (ближний бой, агрессивно гонится) и Демон (держит дистанцию, атакует издали, отступает при сближении). (Enemy.cpp)

## Паттерн проектирования
**Template** — абстрактный класс Ability определяет интерфейс Activate(), конкретные способности переопределяют поведение. (Ability.h, Abilities.cpp)

**Factory** — в GameState::SpawnEnemies() создаются враги по данным из GameDatabase. Данные из enemies.csv → объект Enemy.

## SOLID
- **S**: Renderer, TurnSystem, GameDatabase, LevelGenerator — у каждого одна зона ответственности
- **O**: новые способности и враги добавляются без изменения существующего кода
- **L**: Player/Enemy корректно заменяют Entity, все Ability взаимозаменяемы
- **I**: Item, Ability — минимальные интерфейсы без лишних методов
- **D**: TurnSystem зависит от абстракций (Entity, Room), не от конкретных реализаций

## Запуск
Исполняемый файл и все необходимые библиотеки находятся в папке `x64/Debug/`.
